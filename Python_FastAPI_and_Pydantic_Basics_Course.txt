# Lesson: Overview of FastAPI: What is FastAPI?

Welcome to today's lesson on FastAPI, a modern and powerful Python web framework designed to simplify the process of building APIs. FastAPI has gained rapid popularity in the Python community due to its exceptional performance and developer-friendly features. Today, we'll explore what makes FastAPI stand out and how it can be a game-changer for developers looking to build efficient and robust APIs.

## Introduction to FastAPI

FastAPI is a high-performance web framework for building APIs with Python 3.6+ based on standard Python type hints. Created by Sebastián Ramírez, FastAPI is designed to be easy to use and understand, while also providing the speed and performance of frameworks like NodeJS and Go. It leverages Python's type system to provide automatic data validation, serialization, and documentation generation, making it an excellent choice for developers who want to build APIs quickly and efficiently.

### Key Features of FastAPI

1. **Speed**: FastAPI is one of the fastest Python frameworks available, comparable to NodeJS and Go. This speed is achieved through asynchronous programming and efficient use of Python's type hints.

2. **Developer-Friendly**: With FastAPI, you can build APIs with minimal boilerplate code. It automatically generates interactive API documentation using Swagger UI and ReDoc, making it easier for developers to understand and test the API.

3. **Data Validation and Parsing**: FastAPI uses Pydantic models to handle data validation and parsing. This means you can define data models using standard Python types, and FastAPI will automatically validate and parse incoming request data.

4. **Asynchronous Support**: FastAPI supports asynchronous programming, allowing you to write non-blocking code that can handle many requests simultaneously.

5. **Security and Authentication**: FastAPI provides built-in support for security and authentication, including OAuth2, JWT, and API key authentication.

6. **Extensive Features**: FastAPI includes features like WebSocket support, in-process background tasks, startup and shutdown events, and more.

## Core Mechanics of FastAPI

### Routing and Path Operations

In FastAPI, routing is how HTTP requests are mapped to Python functions. You declare path operations using decorators that specify the HTTP method and the URL path. Here's a simple example:

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"Hello": "World"}
```

In this example, the `@app.get("/")` decorator defines a route for the root URL path (`/`). The `read_root` function returns a Python dictionary, which FastAPI automatically serializes to JSON.

### Pydantic Models

FastAPI uses Pydantic models to define and validate the data structure of requests and responses. This is akin to having a blueprint for the data your API will handle. Here's how you can define a Pydantic model:

```python
from pydantic import BaseModel

class Item(BaseModel):
    name: str
    price: float
    is_offer: bool = None
```

You can then use this model in your API endpoints to validate incoming data:

```python
@app.post("/items/")
def create_item(item: Item):
    return {"item_name": item.name, "item_price": item.price}
```

In this example, the `create_item` function expects a JSON object that matches the `Item` model. FastAPI automatically validates the incoming data and provides a clear error message if the data is invalid.

### Automatic Documentation

FastAPI automatically generates interactive API documentation using Swagger UI and ReDoc. This documentation is available at `/docs` and `/redoc` by default. It allows developers to explore and test the API endpoints without needing to write additional documentation.

## Practical Example: Building a Simple API

Let's build a simple API to manage a list of tasks. We'll define a `Task` model and create endpoints to add, view, and delete tasks.

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List

app = FastAPI()

class Task(BaseModel):
    id: int
    title: str
    completed: bool = False

tasks = []

@app.post("/tasks/", response_model=Task)
def create_task(task: Task):
    tasks.append(task)
    return task

@app.get("/tasks/", response_model=List[Task])
def get_tasks():
    return tasks

@app.delete("/tasks/{task_id}")
def delete_task(task_id: int):
    for task in tasks:
        if task.id == task_id:
            tasks.remove(task)
            return {"message": "Task deleted"}
    raise HTTPException(status_code=404, detail="Task not found")
```

In this example, we define a `Task` model with an `id`, `title`, and `completed` status. We then create endpoints to add a new task, retrieve all tasks, and delete a task by its ID. FastAPI handles data validation and serialization, making it easy to focus on building the API logic.
